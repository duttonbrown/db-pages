When does an order get its own page?

Walk each Shopify line down the diagram. The first Yes decides where it lands. Hover any box for the why.

Decision flow

Excluded SKU?i
Bulbs, return labels, fees, signature upcharges, package protection.
Matches: BULB*, X-*, EXP*, CUSTOM-*, SIG*, or names containing "package protection", "expedite your order", "rework fee", "free return".
YES
Skip
No page. No row.
Not a production item — counted nowhere.
NO
Is it a sample?i
SKU starts with SAC or title contains "sample".
Behavior depends on whether the rest of the order is also samples or has fixtures too.
YES
Shipping (sample-only order)
1 page. No Notion row.
Whole order is samples. One Shipping printout, no Assembly Log row. Pre-packaged from sample inventory.
Held (mixed with fixture)
No page. No row.
Sample shares an order with a fixture. Held in sample inventory; fulfillment grabs it and drops it in the box when the fixture ships.
NO
Title starts with "Color"?i
Color products go through powder coat. They group across SKUs by color — same color on different fixtures = one paint group, one printout. The title prefix is the source of truth (not the SKU).
YES
Powder Coating
Group by color · One Notion row per (SKU, color, finish)
Page key: ("color", color). Same paint color across SKUs = one printout (paints together). Brass-Barely and Black-Barely are different builds, so each gets its own Notion row even on the same paint page.
NO
SKU is a 5-digit fixture or H-prefix hardware?i
Lighting: 5-digit base like 30220, 31000.
Hardware: H + 2 digits like H62-18, H10-2.
YES
Assembly
Group by SKU + finish · One Notion row per (SKU, finish)
Finish-only build packet. Page key: ("finish", SKU, finish). Brass and Nickel are separate builds, so each gets its own row.
NO
Order is accessories-only?i
Default branch. Anything that isn't an excluded SKU, sample, color product, or 5-digit / H-prefix fixture is an accessory: extension rods, bushings, hang-straights.
YES
Shipping (accessories only)
Group by SKU · One Notion row per SKU
Quick-pull from inventory. Page key: ("simple", SKU).
Assembly + flag (mixed order)
Goes with the fixture · Catherine confirms
Accessory shares an order with a fixture. Defaults to Assembly with a TODO note for Catherine to confirm (Shipping vs Assembly). After ~30 cases we'll codify the rule.

How many pages per group?

Default
1 page per group, regardless of qty.
Qty 10 of one fixture in one color → still 1 page. The lead clicks +1 on the queue page to split into duplicates if work needs to be parallelized.
Lead override
Edit the page count directly.
Edits persist in the browser via localStorage — they survive the hourly auto-refresh. A reset link in the page subheader clears them.

How rules improve over time

Step 1
Manager overrides on the queue page.
Click a different Location bubble on any row. Each click writes a log entry the page can export. Survives the hourly refresh.
Step 2
Weekly mine.
A small script surfaces patterns: "This (SKU + finish) was rerouted 5+ times this week." Each candidate is one-click Promote to a real rule.
Step 3
Structural rules from Notion data.
The Notion Parts DB Black Batch flag is already in bom-master.csv. Wire it back into routing → black-finish hardware skips Powder Coat without anyone clicking.
Why human-in-the-loop
~14 colors × 3 finishes × 5 SKU classes = small.
A model is overkill and lets one anomaly flip a rule overnight. A weekly promotion step is fast, debuggable, and matches how Catherine already thinks.
