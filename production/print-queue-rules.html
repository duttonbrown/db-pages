Production · Rules

When does an order get its own page?

Walk each Shopify line through the steps below. Where it lands tells you where it routes and how it groups for printing. Source of truth: work-order-classification.md §8.

The decision, step by step

Is the SKU excluded?

Bulbs (BULB*), return labels (X-*), expedite fees (EXP*), custom-color line items (CUSTOM-*), signature upcharges (SIG*), package protection, "expedite your order", "rework fee", "free return".

Yes→skip No→continue

Is it a sample?

SKU starts with SAC or the title contains "sample".

Sample-only order→Shipping · 1 page · no Notion row Mixed with fixture→held until fixture ships Not a sample→continue

Does the title start with "Color"?

Color products group across SKUs by color (parts paint together). One Notion row per (SKU, color, finish) because Brass-Barely and Black-Barely are different builds even at the same paint job.

Yes→Powder Coating · group key ("color", color) No→continue

Is the SKU a 5-digit fixture or H-prefixed hardware?

Lighting like 30220 or hardware like H62-18. One page per (SKU, finish) — Brass and Nickel are separate builds.

Yes→Assembly · group key ("finish", SKU, finish) No→continue

Otherwise it's an accessory.

Extension rods, bushings, hang-straights, etc. Routing depends on what else is in the order.

Order is accessories-only→Shipping Mixed with fixtures→Assembly + flag for Catherine

How many pages per group?

Default
1 page per group, regardless of qty.
Qty 10 of one fixture in one color → still 1 page. The lead clicks +1 on the queue page to split into duplicates if work needs to be parallelized.
Lead override
Edit the page count directly.
Edits persist in the browser via localStorage — they survive the hourly auto-refresh. A reset link in the page subheader clears them.

Worked examples

A · One color, two finishes
31000-1K (Brass Barely) + 31000-2K (Black Barely)
Both color products in Barely → one paint group. 1 printout · 2 Notion rows.
B · Sample-only order
SAC-PG (Python Green sample)
Short-circuit. 1 printout · Shipping · 0 rows.
C · Sample mixed with fixture
SAC-PG + 31000-1K
Sample is held; fixture follows the color path. 1 printout · 1 row.
D · Two SKUs, same color
30220-2F (47" Slate Blue) + 31000-1F (32" Slate Blue)
Same paint color across SKUs → one page covers both. 1 printout · 2 rows.
E · Accessories only
ER1-3-1 (rod) + BU (bushing)
No fixtures in the order → both go to Shipping. 2 printouts · 2 rows.
F · Accessory + fixture mixed
ER1-3-1 + 31000-1
Rod row → Assembly + flagged for Catherine to confirm. Fixture → Assembly normally. 2 printouts · 2 rows.

How the rules improve over time

Step 1
Manager overrides on the queue page.
Click a different Location bubble on any row. Each click writes a log entry the page can export. Survives the hourly refresh.
Step 2
Weekly mine.
A small script surfaces patterns: "This (SKU + finish) was rerouted 5+ times this week." Each candidate is one-click Promote to a real rule.
Step 3
Structural rules from Notion data.
The Notion Parts DB Black Batch flag is already in bom-master.csv. Wire it back into routing → black-finish hardware skips Powder Coat without anyone clicking.
Why human-in-the-loop
~14 colors × 3 finishes × 5 SKU classes ≈ small.
A model is overkill and lets one anomaly flip a rule overnight. A weekly promotion step is fast, debuggable, and matches how Catherine already thinks. Model later when there's signal to tune.
